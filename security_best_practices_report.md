# Boss Task Agent - 安全最佳实践与漏洞审查报告

**版本**: 1.0  
**审查日期**: 2026-07-16  
**安全等级**: 本地个人/团队工具级安全标准  

---

## Executive Summary (执行摘要)

本报告针对 **Boss Task Agent**（本地客户端与智能体系统）进行代码和架构层面的安全审查。由于本系统主要运行在用户本地环境，无公网暴露的 Web 服务端口，因此整体安全风险较低。

审查的重点在于 **API 密钥存储安全性**、**数据输入防注入 (XSS)**、**二进制分发完整性** 及 **依赖项安全管理**。目前系统已具备优秀的安全基线（如 `.gitignore` 密钥防护和 HTML 看板的 `html.escape` 字符转义），本报告对潜在的本地泄露风险及未来系统扩展提出了前瞻性的安全优化建议。

---

## 1. 密钥管理与凭据安全 (Secrets Management)

### [ID-01] [中危] 敏感凭据本地明文存储风险
- **影响陈述**: 任何能访问本地系统的第三方用户或恶意软件都可以直接读取明文的 API Key，造成大模型账号额度被盗刷。
- **关联代码**: [config.py:47-49](file:///D:/项目ai/boss/config.py#L47-L49), [gui.py:545-552](file:///D:/项目ai/boss/gui.py#L545-L552)
- **现状分析**: 
  系统目前将 API Key 写入同级目录下的 `.env` 明文文件中，虽然通过 `.gitignore` 规避了提交到 Git 仓库的风险，但在本地主机上，该文件仍为任何本地用户可读。
- **优化建议**:
  1. **系统权限隔离**: 在 Windows 上，对 `.env` 文件右键属性 -> 安全，设置仅当前 Windows 登录用户有读取权限，阻止其他本地用户或共享网络用户访问。
  2. **企业级升级方案**: 如果未来部署到服务器或企业级环境，应改用系统环境变量或对接密钥管理系统（如 HashiCorp Vault、AWS Secrets Manager）。

---

## 2. 输入注入防护 (Injection Defense)

### [ID-02] [低危] 静态 HTML 进度看板跨站脚本攻击 (XSS)
- **影响陈述**: 恶意构造的 Excel 语音数据或 LLM 异常输出可能包含 HTML/JS 脚本，导致在浏览器打开看板时执行非法本地脚本。
- **关联代码**: [dashboard_generator.py:12](file:///D:/项目ai/boss/dashboard_generator.py#L12)
- **现状分析**:
  系统在构建 HTML 看板时，已前瞻性地引入了 `html.escape` 过滤函数，并将所有来自 Excel 或大模型的变量（如任务标题、描述、交付物名称等）包裹在 `esc()` 中进行字符转义。这已有效防御了常规 XSS 攻击。
- **优化建议**:
  - 维持当前策略。未来若在 HTML 看板中添加交互式输入框或引入外部 JS 库（如甘特图），需继续确保数据流经过滤转义，防范 DOM-based XSS。

## 3. 二进制编译与分发安全 (Distribution Integrity)

### [ID-03] [低危] PyInstaller 二进制免杀与防篡改
- **影响陈述**: 生成的 `BossTaskAgent.exe` 可能会被 Windows Defender 等杀毒软件误报，或在分发给其他同事时被第三方中途篡改植入恶意代码。
- **关联代码**: [build.bat:21-38](file:///D:/项目ai/boss/build.bat#L21-L38)
- **现状分析**:
  系统通过 PyInstaller 将 Python 脚本打包为 exe，这在没有代码签名的情况下，容易在其他电脑上触发安全阻拦警告。
- **优化建议**:
  1. **哈希校验**: 在打包完成发布时，生成该 `.exe` 文件的 SHA-256 哈希值并与程序一同发布。用户下载后可以通过命令行校验哈希值确保文件未被篡改。
     *   校验命令: `Get-FileHash .\BossTaskAgent.exe`
  2. **加壳/签名**: 企业内部发布时，建议使用企业自签名的代码证书对生成的 `.exe` 进行签名，消除杀毒软件的误报警告。

---

## 4. 第三方依赖包安全 (Dependency Security)

### [ID-04] [低危] 第三方库漏洞暴露面
- **影响陈述**: 使用的底层开源库（如 `openpyxl`、`pydantic` 等）若存在历史已知漏洞，可能被恶意的 Excel 文件触发 XML 外部实体注入 (XXE) 或拒绝服务攻击。
- **现状分析**:
  当前项目依赖直接写在 `requirements.txt` 中，未锁定具体安全子版本。
- **优化建议**:
  - 定期运行 `pip list --outdated` 或使用 `safety` 工具检查依赖安全漏洞。
  - 在 `requirements.txt` 中锁定经过安全验证的库版本（例如：`pydantic>=2.0.0`）。
